# 📍 useCallback

#### 2026. 04. 02.

<br>

### 💡 useCallback이란?

함수 자체를 메모이제이션하여 자식 리렌더 방지
콜백 함수를 메모이제이션하고 재사용할 수 있도록 하는 훅
주로 부모 컴포넌트로부터 자식 컴포넌트에 전달되는 콜백 함수를 최적화하는데 사용

```js
useCallback(() => {
  // 함수 본문
}, [dependency1, dependency2]);
```
`함수 본문`: 메모이제이션할 함수

`[dependency...]`: 의존성 배열에 포함된 값들이 변경될 때마다 새로운 함수를 생성 | 변경되지 않으면 이전에 생성된 함수를 변환

<br>

## 📃 useCallback을 사용해야 하는 경우

### ☝️. 이벤트 핸들러의 재사용 (메모리 효율)
```
👩‍💻 매번 새로 만들기 x, 만들어둔 걸 재사용
```

리액트 컴포넌트는 상태가 변할 때마다 함수 전체가 다시 실행된다. 이때 내부의 선언된 함수들도 매번 **새로** 만들어진다.

#### 🚨 | 문제점
똑같은 함수인데, 컴퓨터 메모리 입장에서는 매번 새로운 주소에 함수를 생성해야함

  → 메모리 자원 낭비

### useCallback 활용
첫 렌더링 때 함수를 메모리 전용 창고(캐시)에 저장한다.

다음 랜더링 때는 창고에서 꺼내 쓴다.

> 🤩 | **불필요한 생성 / 삭제 과정이 사라진다!!** 

---

<br>

### ✌️. 하위 컴포넌트의 불필요한 재렌더링 방지
리액트 자식 컴포넌트는 `props`가 변했다고 판단되면 자신을 다시 그린다.

리액트는 함수를 비교할 때 내용이 아닌 **참조값(메모리 구조)**을 본다.

`useCallback`을 쓰면 함수의 주소값이 그대로 전달되면서 자식 컴포넌트가 그걸 무시할 수 있게 되면서 재렌더링을 방지할 수 있다!

---

<br>

### 👌. 렌더링 최적화가 필요한 경우

useCallback을 사용하여 함수를 재사용하면, 렌더링 최적화를 수행할 수 있다. 이는 컴포넌트의 불필요한 재렌더링을 방지하고, 성능을 향상시킬 수 있다.

---

### 💡 요약 정리
> `useCallback`은 함수를 필요할 때 꺼내 씀으로써, 리액트가 함수가 랜더링 됬다며 자식 컴포넌트들도 다시 재 랜더링하는 걸 방지하기 위한 도구이다!

**주의 할 점:** 모든 함수에 `useCallback`을 남발하면 오히려 비용이 더 들 수 있다...

**자식 컴포넌트에 props로 넘겨주는 함수** 또는 **다른 Hook의 의존성 배열에 들어가는 함수** 위주로 사용하는 것이 가장 효율적이다!

## 💭 useMemo와의 관계?
둘은 본질적으로 같은 메커니즘이다.
``` js
const sorted = useMemo(() => {
  return list.sort();
  // 함수를 실행한 결과 값
}, [list]);
// sorted = [1, 2, 3, ...] 값 형식
```

``` js
const sort = useCallback(() => {
  return list.sort();
  // 함수 자체를 저장
}, [list]);
// sort = () => {...} 함수
```
내부 구현은 동일하지만

`useMemo`는 콜백을 실행해서 그 결과를 저장(캐싱)

`useCallback`은 콜백 실행 ✖️, 그 자체를 저장(캐싱)
