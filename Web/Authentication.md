# 세션 | 토큰

## 세션 (Session)

### 실생활 비유

- 카페 직원이 손님의 얼굴을 기억

<br>

> 로그인하면 서버가 "이 사람 로그인했다."고 기억하고, 브라우저 세션 ID(일종의 번호표)만 쿠키로 준다.
> 이후 요청할때마다 번호표를 내밀면 서버가 확인하는 방식

- 브라우저를 닫으면 사라짐
- 서버가 모든 정보를 들고 있어서 부담이 커짐
- 가장 전통적인 방식

## 토큰 (Token / JWT)

### 실생활 비유

- 회원권 카드를 내가 보관

<br>

> JWT(JSON Web Token)가 대표적인데 나는 oo이고 관리자야라는 정보가 암호화돼서 토큰 안에 들어있음, 서버는 그걸 복호화해서 확인만 하면 된다.

- 서버가 아무것도 기억 안해도 됨 -> 서버 확장에 유리
- 앱, 웹, 다른 서버 등 어디서든 같은 토큰 사용 가능
- 요즘 API 인증에서 가장 많이 쓰는 방식
