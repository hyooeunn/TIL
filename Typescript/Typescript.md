# 📌 TypeScript (타입스크립트)
## 2025. 11. 10. 월요일

## Typescirpt란?
- 자바스크립트의 슈퍼셋(Superset)
- 자바스크립트에 __타입__ 을 부여한 언어

<br>

ts -> js -> 코드 실행

## Typescript를 사용하는 이유?
- JS는 런타임에서 타입이 결정되는 동적 언어
  - 즉 런타임에서만 오류를 발견할 수 있게 되므로 __타입 안정성__ 이 보장 ❌
  - 큰 규모의 프로젝트 일수록 디버깅이 어려움!

## JS와 TS의 차이
### JavaScript
- 언어의 변환 ❌, __인터프리터__ 언어

### TypeScript
- 컴파일러 필요 ✅, 변환이 필요한 __컴파일러__ 언어


## Typescript의 특징
### 1. 타입스크립트는 컴파일 언어이자, 정적인 타입(Static Type)
  - 코드의 가독성을 높여줌
  - 자바스크립트의 단점인 타입 안정성을 보장해준다
  - 기업에서 많이 쓴다 취업하기 위해선 꼭 필요함 그냥

### 2. 객체지향 프로그래밍 언어를 지원한다
  - __ES6 문법__ 을 포함하며, 클래스, 인터페이스, 상속, 모듈 등과 같은 __객체 지향 프로그래밍__ 패턴을 제공
