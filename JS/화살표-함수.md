# 📍 화살표 함수 (Arrow function)

- JS에서 사용하던 기존의 함수 선언식이나 함수 표현식보다 좀 더 간결하며, 쉽게 사용하기 위해 만들어졌다.

## 기본 문법

- `function` 키워드 대신 `=>`를 사용한다.
- 매개변수가 하나면 `()` 생략 가능하다.
- 본문이 한 줄이면 `{}`와 `return` 생략 가능하다.
  - `return` 키워드를 생략할 경우 반드시 `{}`도 생략해야합니다.
- 중복된 매개변수 이름을 선언할 수 없습니다.

#### 일반함수

```js
const add = function (a, b) {
  return a + b;
};
```

#### 화살표 함수

```js
const add = (a, b) => {
  return a + b
};
-
const add = (a, b) => a + b;
```

`function` 키워드 대신 `=>`를 사용해 기존 함수보다 더 간결하게 작성할 수 있다.

<br>

## 사용하는 이유

- 코드가 간결해지고 가독성이 좋아짐
  - 특히 map, filter에서 유용함

<br>

## 일반 함수와의 차이점

- 화살표 함수는 **인스턴트**를 생성할 수 없습니다.
  - 즉, 생성자 함수로 사용할 수 없습니다.
- 화살표 함수는 자신만의 **this**를 가지지 않습니다.
- 화살표 함수는 **arguments** 객체를 가지지 않습니다.

<br>

### 화살표 함수는 자신만의 `this`를 가지지 않는다.

- `this`: 지금 이걸 실행하고 있는 주체 (나를 가르키는 것)

this 사용 법

```js
const hyoeun = {
  name: "효은",
  greet: function () {
    console.log(this.name); // 여기서 this = 효은
  },
};

hyoeun.greet(); // "효은" 출력
```

this의 문제

```js
const hyoeun = {
  name: "효은",
  greet: function () {
    setTimeout(function () {
      conosole.log(this.name);
      // 여기서 this는 효은이 아님 ✖️
    }, 10000);
  },
};
```

> `setTimeout` 안의 함수는 철수가 아니라 타이머가 호출된다.
>
> - 즉 this가 갑자기 타이머를 가르켜 버린다.

<br>

#### 화살표 함수는 `this`를 바꾸지 않는다.

- 자기가 정의된 위치의 this를 평생쓴다.

```js
const hyoeun = {
  name: "효은"
  greet: function() {
    setTimeout(() => {
      console.log(this.name) // this = 효은
    }, 1000);
  }
};
```

화살표 함수`(() =>)`는 `this`를 신경쓰지 않고 위에 있는 `this`를 그대로 빌려온다.

| |일반 함수|화살표 함수|
|---|---|---|
|`this`가 언제 정해지냐|호출될때 마다 새로 정해짐|정의될 때 고정|
비유|대화 상대에 따라 "나"가 달라짐|항상 같은 "나"를 씀
